## Standard Operating Procedures

### SOP-001: Tone Injection & Depth Audit
**Trigger:** Receives a content brief from @Marcus.

1. Run Session Start Protocol â€” check `CLINE_SYNC.md` and `active_context.md`
2. Query Per-Client Tone Profile for the target project
3. Analyze existing draft for "flatness" or generic "vanilla" claims
4. Inject the "Beast-Mode" narrative (Luxury standards, cinematic flow)
5. Verify tone matches the client's brand guide
6. Cross-reference every claim with verified facts
7. Update content depth score

### SOP-002: Truth-Lock Sign-Off
**Trigger:** Page or feature is ready for final content review.

1. Scan for placeholder text: Lorem Ipsum, "artisan", "homemade" (without evidence), generic claims
2. Cross-reference every claim with the project's Truth Core
3. Verify all images are REAL client assets (not stock photos of people)
4. Check that testimonials are from real customers (or clearly marked as examples)
5. If any claim is unverified â†’ BLOCK the Truth Gate
6. **Database Saturation**: Verify >100 real entries for content-driven modules (no skeletons)
7. If 100% verified â†’ Update Sign-Off table

### SOP-003: Placeholder Detection Protocol (NEW)
**Trigger:** Before ANY deployment, or after any multi-file edit session.

**The INC-001 Rule:** After the La-Aesthetician incident where placeholder text went live, this is now mandatory.

| Scan Target | Detection Pattern | Action |
|:------------|:-----------------|:-------|
| Lorem Ipsum | `grep -ri "lorem\|ipsum\|dolor sit" src/` | BLOCK deploy |
| Generic claims | "world-class", "best in class", "artisan" without evidence | Flag for rewrite |
| TODO comments | `grep -rn "TODO\|FIXME\|PLACEHOLDER" src/` | Must resolve before deploy |
| Stock photos | Any image of people not from client's own photography | BLOCK deploy |
| Empty sections | Components with no real content | BLOCK deploy |
| Default text | "Your Company Name", "example@email.com" | BLOCK deploy |

**Trigger:** New client onboarding, or content review.

Maintain a distinct voice for each project:

### SOP-005: Content Preservation Protocol (NEW)
**Trigger:** Any multi-file edit session, especially when @Sebastian or @Priya are refactoring.

**The Golden Rule:** Real content must NEVER be overwritten with placeholder text during code edits.

1. Before any edit: screenshot or copy the current live content
2. After any edit: diff the content â€” did any real text disappear?
3. If content was lost: BLOCK the commit and restore from backup
4. Flag to @Marcus if content preservation was violated
5. Log in incident tracker

### SOP-006: Brand Guide Compliance Audit (NEW)
**Trigger:** Before any client deployment, or when new pages are added.

1. Check project has a `BRAND_GUIDE.md` â€” if not, create one
2. Verify color palette matches brand guide (coordinate with @Priya)
3. Verify typography matches brand guide
4. Verify imagery style matches brand guide
5. Verify tone matches Per-Client Tone Profile (SOP-004)
6. Check for consistency across all pages (no tone drift)
7. Update Brand Compliance score
