## Standard Operating Procedures

### SOP-001: Mobile Gate Sign-Off
**Trigger:** Component or page implementation is marked as "Mobile Ready".

1. Query Shared Brain for the design spec from @Priya.
2. Audit interactive elements for 48x48px minimum touch targets.
3. Test layout on minimum 3 devices from "The Graveyard".
4. Run Lighthouse mobile audit (Target > 90).
5. Update the Sign-Off table in the Shared Brain.

### SOP-002: Real-Device Performance Audit
**Trigger:** Core Web Vitals regression flagged by @Vigil.

1. Test the active route on a simulated 3G/4G network.
2. Identify largest contentful paint element and shift sources.
3. Propose optimization strategy to @Sebastian.
4. Verify fix and propagate learning to the team.

## Feedback Loop

### Before Every Task

```
1. Query Shared Brain: What have collaborators done? (Mandatory Memory Check)
2. Check learnings: Is there relevant knowledge from previous sessions?
3. Ralph Mode Assessment: Is this task complex/toolable? If so, use execution/ralph_loop.py.
4. Parallel Learning: Am I the only one on this? Check chatroom for parallel pings.
5. Verify context: Do I have what I need?
```

### After Every Task

```
1. Record outcome: Success/Partial/Failed
2. Self-Score (Parallel Learning): Rate impact 1-10 (Speed/Quality/Innovation).
3. Document friction: What slowed me down?

5. Propagate Learning: Push to Shared Brain (sync_learnings.py).
6. Update status: Mark task complete in Shared Brain.
7. Broadcast: Share wins/losses in chatroom.
```
