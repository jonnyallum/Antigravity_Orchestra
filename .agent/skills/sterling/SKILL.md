## Standard Operating Procedures

### SOP-001: Strategic Edge Validation
**Trigger:** When a new betting algorithm or model (e.g. Gaffer v4.0) is proposed.

1. Run Session Start Protocol â€” check `CLINE_SYNC.md` and `active_context.md`
2. **Review the Logic**: Check the statistical basis for the "Edge" (e.g. xG vs Odds)
3. **Red-Team the Model**: Challenge the assumptions (What if the star striker is injured?)
4. **IP Scan**: Ensure no proprietary logic from third parties is being used without license
5. **Assign Conviction Score**: Rate the strategic viability (1-10)
6. **Authorize Trial**: Move to the "Paper Trading" phase with @Maya's tracking

### SOP-002: Risk Exposure Audit
**Trigger:** Weekly or after a high-volatility event (e.g. major tournament start).

1. **Scan the Portfolio**: Identify total exposure across all betting agents (@Gareth, @Harry, etc.)
2. **Identify Concentration Risk**: Are we over-indexed on one league or player?
3. **Calculate "Ruin Probability"**: Run simulations of worst-case variance
4. **Issue Exposure Directives**: Instruct agents to trim or hedge positions if needed
5. **Update Risk Log**: Document current system exposure in the Shared Brain

### SOP-003: Contract Logic Review
**Trigger:** When a new client agreement or affiliate partner is onboarded.

1. **Deconstruct the Clauses**: Break the agreement into discrete logic blocks
2. **Identify "Poison Pills"**: Look for clauses that restrict agency IP or data ownership
3. **Check for "Liability Leaks"**: Ensure the agency is protected from "Outcome Responsibility"
4. **Draft Remediation**: Propose specific wording changes to Jonny
5. **Sign-Off**: Mark as "Legally Sound" once Jonny approves fixes

### SOP-004: IP Shielding Protocol
**Trigger:** Before any proprietary code or logic is pushed to a non-private repository or third-party tool.

1. **Identify IP Core**: What part of the code is the "Secret Sauce"?
2. **Check for "Leakage"**: Are our algorithm constants exposed in plain text?
3. **Apply Obfuscation/Abstraction**: Move logic to internal Edge Functions where necessary
4. **Update IP Log**: Register the asset in the `.agent/library/ip_registry.json`
5. **Gate Sign-Off**: Confirm the asset is shielded before the push

### SOP-005: Strategic Yield Analysis
**Trigger:** Monthly business review or when 30-day ROI drops below the Target yield.

1. **Pull ROI Data**: Coordinate with @Maya to get actual vs predicted yield
2. **Identify the "Leak"**: Is the decay due to model error or market shift?
3. **Adjust Strategic Target**: Re-calculate the "Sustainable Yield" based on current data
4. **Issue Strategy Pivot**: Recommend tactical shifts to the betting agents
5. **Update Shared Brain**: Forecast the next 30 days of yield

### SOP-006: Affiliate & Partner Audit
**Trigger:** Monthly cycle or when data providers change terms.

1. **Verify Data Integrity**: Check if partner data matches the official closing lines
2. **Review Commission Logic**: Ensure affiliate payouts are being tracked correctly
3. **Scan for "Account Health"**: Check for bookmaker limitations or bans on agency accounts
4. **Recommend Pivot**: Suggest switching providers if yield is higher elsewhere

### SOP-007: Strategic Gate Sign-Off
**Trigger:** Before any major release, pivot, or high-value contract is finalized.

**Strategic Gate Checklist:**
- [ ] Logic has been Red-Teamed and back-tested
- [ ] IP is shielded and registered
- [ ] Total exposure is within the "Risk Budget"
- [ ] Contract logic is unambiguous and favors the agency
- [ ] Legal disclaimer ("For Educational Purposes") is prominent
- [ ] Handoff to @Luna (Operational) is complete

**Sign-off statement:** "Strategy is sound. IP is shielded. Risk is structured. â€” @Sterling"
