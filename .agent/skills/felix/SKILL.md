### SOP-001: Monetization Strategy Audit
**Trigger:** Start of a new project or major feature expansion.

1. Run Session Start Protocol ‚Äî check `CLINE_SYNC.md` and `active_context.md`
2. Query project brief for target audience and market positioning
3. Research competitor pricing (minimum 3 competitors)
4. Outline the core monetization engine:

| Model | Best For | Examples |
|:------|:---------|:--------|
| SaaS/Subscription | Recurring value delivery | Kwizz Pro, Insydetradar Premium |
| Freemium | User acquisition ‚Üí conversion | Kwizz (free quizzes ‚Üí paid packs) |
| One-Time Purchase | High-value deliverables | Agency builds, website packages |
| Lead Generation | Service businesses | DJ Waste, CD Waste, La-Aesthetician |

5. Draft monetization spec: Pricing tiers, offer stack, upsells, funnel
6. Align with @Elena on sales copy and @Grace on search intent
7. Present to @Marcus for approval before development begins

### SOP-002: Pricing Tier Design (NEW)
**Trigger:** Any project that needs a pricing page or subscription model.

**The Antigravity Pricing Framework:**

| Tier | Purpose | Pricing Psychology |
|:-----|:--------|:------------------|
| Free/Starter | User acquisition, social proof | Remove friction, build habit |
| Pro/Growth | Core revenue driver | Must feel like "obvious value" |
| Enterprise/Premium | High-margin, low-volume | White-glove, custom features |

**Rules:**
1. Always have 3 tiers (anchoring effect)
2. Middle tier should be the "recommended" option
3. Annual pricing = monthly √ó 10 (2 months free)
4. Feature gating must feel fair ‚Äî don't cripple the free tier
5. Price based on VALUE delivered, not cost to serve
6. Include social proof on pricing page (coordinate with @Elena)

### SOP-004: ROI Sign-Off (NEW)
**Trigger:** Feature completion or scheduled project review.

1. Audit implementation against the original monetization goal
2. Verify tracking and attribution logic
3. Confirm checkout/conversion paths are frictionless
4. Calculate projected ROI:
   ```
   Monthly Revenue = (Traffic √ó Conversion Rate √ó Average Order Value)
   CAC = Total Marketing Spend / New Customers
   LTV = Average Revenue Per User √ó Average Lifespan
   ROI = (LTV - CAC) / CAC √ó 100
   ```
5. Update the Revenue Gate sign-off
6. Report to @Marcus with revenue projections

### SOP-005: Competitor Pricing Intelligence (NEW)
**Trigger:** Before any pricing decision, or quarterly review.

1. Identify top 5 competitors in the project's market
2. Document their pricing:
   - Tier names and prices
   - Feature breakdown per tier
   - Free trial / freemium offering
   - Annual vs monthly pricing
3. Map our positioning:
   - Are we premium, mid-market, or budget?
   - What's our unique value vs competitors?
   - Where can we undercut or differentiate?
4. Create pricing comparison table
5. Share with @Marcus and @Elena for positioning alignment

### SOP-006: Payment Integration Strategy (NEW)
**Trigger:** Project needs to accept payments.

| Platform | Payment Method | Best For |
|:---------|:-------------|:---------|
| Stripe | Web payments | SaaS, subscriptions, one-time |
| Supabase + Stripe | Auth + billing | Full-stack web apps |
| RevenueCat | In-app purchases | Mobile apps (coordinate with @Blaise) |
| Stripe Connect | Marketplace | Multi-vendor platforms |

**Integration Checklist:**
1. Choose payment provider based on platform
2. Design pricing schema (coordinate with @Diana for database)
3. Implement checkout flow (coordinate with @Sebastian)
4. Test with Stripe test mode before going live
5. Set up webhook handlers for subscription events
6. Implement grace period for failed payments
7. Create customer portal for subscription management
8. Verify PCI compliance (coordinate with @Sam)

### SOP-007: Monetization Health Check (NEW)
**Trigger:** Monthly, or when revenue targets are missed.

| Metric | Formula | Target | Action if Below |
|:-------|:--------|:-------|:---------------|

| LTV:CAC Ratio | LTV / CAC | > 3:1 | Reduce acquisition cost |
